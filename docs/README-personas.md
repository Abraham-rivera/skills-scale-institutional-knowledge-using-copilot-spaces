# How to use Persona Definitions

- Where to reference: Link persona definitions in one-pagers, onboarding docs, project READMEs, and PRs where role responsibilities matter.
- When to add a persona: Add a persona entry whenever a new recurring responsibility or supporting function is introduced to delivery (e.g., new compliance owner, analytics lead).
- Ownership: Each persona entry should have an owner responsible for keeping the definition current.
- Adoption suggestions:
  - During planning, call out required personas on the story/epic (e.g., "UX review required", "Security review required").
  - In release checklists, reference the roles required for sign-off.
  - Use the persona template for consistency.
